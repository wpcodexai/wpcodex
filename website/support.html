Common issues

"Cannot connect to MCP server" error
Check that (1) WPCODEX_ENABLE_MCP is defined and true in wp-config.php, (2) the REST API is accessible (not blocked by a security plugin or WAF), (3) your credentials are correct, and (4) the application password hasn't expired or been revoked.
HTTP 403 Forbidden on all ability calls
The authenticating user needs the manage_options capability. Ensure your WordPress account is an Administrator. Also check that Application Passwords haven't been disabled by a security plugin (e.g., some configurations of Wordfence or All In One WP Security disable them).
WP-CLI commands return "command not found"
WP-CLI must be installed and accessible on the server's PATH for the web server process. Some shared hosts restrict CLI access. If WP-CLI isn't available, all other abilities still work over HTTP.
Skills aren't loading at session start
Skills with enable_agentic: true are loaded by the agent, not by WPWorker automatically. Your agent's system prompt or session protocol must call wpworker/skill-list and then wpworker/skill-read for each agentic skill. See the Session Start Protocol in the docs.
PHP execute returns a blank result
Your code must explicitly return a value. PHP code that only echoes output will return an empty string. Use return to pass data back: return get_bloginfo('name');
