Quick Start

Up and running in 5 minutes

Follow these steps and your AI agent will have full WordPress control before your coffee's cold.

1

Download & install the plugin

Upload the ZIP in WordPress Admin → Plugins → Add New → Upload Plugin, then activate. Or use WP-CLI:

wp plugin install wpcodex --activate
2

Enable MCP in wp-config.php

Open wp-config.php and add this line before "That's all, stop editing!":

define( 'WPCODEX_ENABLE_MCP', true );

Without this constant the plugin is completely dormant — no REST endpoints, no MCP server.

3

Create an Application Password

In WordPress Admin, go to Users → Your Profile → Application Passwords. Enter a name (e.g. "Claude Agent") and click Add New Application Password. Copy the generated password — it won't be shown again.

4

Note your MCP endpoint URL

Your endpoint is:

https://yoursite.com/wp-json/wpcodex/v1/mcp

Replace yoursite.com with your actual domain. The site must be accessible over HTTPS from your agent's network.

5

Connect Claude Desktop

Edit your Claude Desktop config file (claude_desktop_config.json) and add:

{
  "mcpServers": {
    "wpcodex": {
      "url": "https://yoursite.com/wp-json/wpcodex/v1/mcp",
      "headers": {
        "Authorization": "Basic <base64(admin:app-password)>"
      }
    }
  }
}

Encode username:application-password in Base64. On Mac/Linux: echo -n "admin:xxxx xxxx" | base64

6

Test the connection

Restart Claude Desktop. In a new conversation, ask Claude: "Call wpcodex/site-info and tell me what WordPress version I'm running."

If Claude returns your site's WP version, you're connected! 🎉

Using Cursor or another MCP client?

The connection details are the same — URL and Basic auth header. Check your client's MCP documentation for where to configure MCP servers. The full docs have client-specific examples.
