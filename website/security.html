Security

Built with security first

WPWorker gives AI agents real power. Here's exactly how that power is constrained, authenticated, and audited.

Security model

Multiple independent layers — every request must pass each one.

🔒

Disabled by default

The MCP server, all REST endpoints, and all abilities are completely dormant unless you explicitly add define('WPCODEX_ENABLE_MCP', true) to wp-config.php. Installing the plugin alone exposes nothing.

🔑

WordPress Application Passwords

Authentication uses the built-in WordPress Application Passwords system (WP 5.6+). Every request requires a valid Authorization: Basic header. Tokens can be revoked instantly from the WordPress admin — no database changes needed.

🛡️

Capability check on every request

Every ability's permission_callback calls current_user_can('manage_options'). Non-admin application passwords cannot call any WPWorker ability, even if they authenticate successfully.

🌐

HTTPS strongly recommended

WPWorker itself does not enforce HTTPS, but your WordPress install should. Application Password credentials travel in the Authorization header — always serve your MCP endpoint over TLS. Most managed WordPress hosts enforce HTTPS automatically.

⚡

Atomic file writes with backups

The wpworker/file-write ability creates a .bak backup before every write. If the agent writes bad content, restore from the backup. No silent overwrites.

📋

Audit log (Pro & Agency)

Every MCP call — ability name, parameters, authenticated user, timestamp, and result status — is logged. Replay or review any agent session. Available on Pro and Agency plans.

Best practices

  • ✓ Create a dedicated WordPress admin account for your AI agent. Name it something obvious like ai-agent so you can audit its activity separately.
  • ✓ Use a unique Application Password per AI client. Revoke individual passwords if a client is compromised without affecting others.
  • ✓ Enable HTTPS on your site before activating WPWorker MCP. Credentials in plain HTTP can be intercepted.
  • ✓ Restrict the MCP endpoint to known IP ranges using your server config or a WAF if your agent runs from a fixed IP.
  • ✓ Stage dangerous changes first. Your AI agent can execute arbitrary PHP — run it on a staging site before production for risky operations.
  • ✓ Review skills regularly. Skills with enable_agentic: true load automatically and influence agent behavior. Audit them like code.
  • ✓ Enable the audit log (Pro+) so you have a complete record of every agent action for accountability and debugging.

Security disclosures

Found a vulnerability? Please report it responsibly to security@wpworker.com. We aim to acknowledge reports within 48 hours and publish advisories in the changelog after fixes are shipped. We do not currently offer a bug bounty but will credit researchers publicly with permission.
