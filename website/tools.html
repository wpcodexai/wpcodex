MCP Abilities

All WPCodex Tools

Every ability registered by WPCodex, ready to be called by your AI agent via the Model Context Protocol.

Core Execution
⚡
wpcodex/php-execute

PHP Execute

Run arbitrary PHP code inside the live WordPress process. Access all globals, hooks, functions, and $wpdb exactly as WordPress itself does. No opening <?php tag needed.

code* (string)
🖥️
wpcodex/wpcli-run

WP-CLI Run

Execute any WP-CLI command on the server without leading wp. Supports an optional timeout. Returns combined stdout/stderr output from the command.

command* (string) timeout (int, seconds)
🗄️
wpcodex/db-query

Database Query

Run SQL via $wpdb with optional prepared statement args. SELECT queries return an array of row objects; INSERT/UPDATE/DELETE return the affected row count.

sql* (string) args (array)
Filesystem
📖
wpcodex/file-read

File Read

Read the contents of any file on the server by absolute path. Returns the raw file content as a string. Useful for inspecting theme files, plugin source, and config.

path* (string)
✏️
wpcodex/file-write

File Write

Atomically write content to any file path. A .bak backup of the original is created automatically before every write. Directories are created if they don't exist.

path* (string) content* (string)
📂
wpcodex/file-list

File List

List the contents of a directory. Optionally recurse into subdirectories. Returns an array of file and directory paths with metadata.

path* (string) recursive (bool)
WordPress Data
🌐
wpcodex/site-info

Site Info

Returns a full snapshot of the WordPress installation: WP version, PHP version, active plugins (with versions), active theme, site URLs, and server environment details.

No arguments required
🔧
wpcodex/option-get

Option Get

Retrieve any WordPress option by name using get_option(). Returns the deserialized value — arrays, objects, and scalar types all work correctly.

option_name* (string)
⚙️
wpcodex/option-set

Option Set

Set any WordPress option by name using update_option(). Accepts complex values (arrays, objects). Optional autoload flag controls whether the option loads on every request.

option_name* (string) option_value* (mixed) autoload (bool)
📝
wpcodex/post-query

Post Query

Run a WP_Query with any query args object. Returns found_posts (total matching) plus a posts array with full post objects and meta.

query_args* (object)
Skills
📚
wpcodex/skill-list

Skill List

Returns all stored skills with their name, description, and flags. Agents call this at session start to discover available playbooks and standing instructions.

No arguments required
📖
wpcodex/skill-read

Skill Read

Load the full Markdown body of a skill by name. Agents call this for agentic skills at session start, and for task-specific skills once the task is clear.

name* (string)
✍️
wpcodex/skill-create

Skill Create

Create a new skill in the database. Supply a name, description, YAML+Markdown body, and optional flags for agentic loading and prompt display.

name* (string) description* (string) body* (string) enable_agentic (bool) enable_prompt (bool)
🔄
wpcodex/skill-update

Skill Update

Update an existing skill. Pass the skill name and any fields to change — description, body, or flags. Partial updates are supported.

name* (string) description, body, enable_agentic, enable_prompt
🗑️
wpcodex/skill-delete

Skill Delete

Permanently delete a skill from the database by name. This action is irreversible — export the skill body first if you need a backup.

name* (string)
