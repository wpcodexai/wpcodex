What's new in WPWorker

Every release, documented. Subscribe to the newsletter for announcements.

v1.0.0 — First Stable Release

New
June 13, 2026
  • Initial stable release of WPWorker
  • Registered 10 core MCP abilities: php-execute, wpcli-run, db-query, file-read, file-write, file-list, site-info, option-get, option-set, post-query
  • Skills system: create, read, update, delete, and list playbooks stored in the database
  • Agentic skills (loaded at session start) and prompt skills (surfaced in AI interfaces)
  • WordPress Application Password authentication on all MCP endpoints
  • manage_options capability check on every ability
  • Atomic file writes with .bak backup on file-write
  • Disabled by default — requires WPCODEX_ENABLE_MCP constant in wp-config.php
  • WordPress Abilities API integration via wordpress/mcp-adapter
  • wp_codex_abilities filter for third-party ability registration
  • PHPStan level 8 compliance throughout
  • Full PHPUnit test coverage for all abilities and runners
  • Admin UI: Skills management page, settings, and MCP connection info

v0.9.0 — Release Candidate

Fix Perf
May 30, 2026
  • Fixed: file-write could overwrite without creating a backup if target directory was non-writable
  • Fixed: wpcli-run timeout not applied when WP-CLI path was resolved via PATH environment
  • Improved: db-query now returns column names even on empty result sets
  • Performance: site-info response cached per-request to avoid redundant filesystem calls
  • Added: wpcodex_after_activate and wpcodex_after_deactivate action hooks
  • Docs: added ability reference and Skills system docs to README

v0.8.0 — Skills System Beta

New
May 10, 2026
  • Introduced Skills system: database-backed Markdown playbooks for agents
  • Added skill-list, skill-read, skill-create, skill-update, skill-delete abilities
  • Skills admin page added to WordPress admin menu
  • enable_agentic and enable_prompt flags on skills

v0.5.0 — Core Abilities Alpha

New Breaking
April 18, 2026
  • First working implementation of core abilities via WordPress Abilities API
  • MCP transport over WordPress REST API (/wp-json/wpworker/v1/mcp)
  • Breaking: renamed ability namespace from novamira/* to wpworker/*
  • Authentication switched from API keys to WordPress Application Passwords
  • Added WPCODEX_ENABLE_MCP constant guard
