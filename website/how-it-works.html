1

Install the plugin

Upload WPCodex to your WordPress plugins directory and activate it. The plugin requires WordPress 6.9+ and PHP 8.0+. It boots no services and exposes nothing by default.

# via WP-CLI
wp plugin install wpcodex --activate
2

Enable MCP in wp-config.php

Add a single constant to wp-config.php. This is the only switch — when absent, the plugin is completely dormant. When present, it registers the MCP transport and all abilities.

// wp-config.php
define( 'WPCODEX_ENABLE_MCP', true );
3

Create a WordPress Application Password

Go to Users → Profile in the WordPress admin, scroll to Application Passwords, and create a new one for your AI agent. Copy the password — you'll need it to connect.

# MCP server URL
https://yoursite.com/wp-json/wpcodex/v1/mcp

# Authentication
Authorization: Basic <base64(user:app-password)>
4

Connect your AI agent

Point your MCP-compatible agent (Claude, Cursor, Cline, Continue, etc.) at the WPCodex endpoint with your credentials. The agent discovers all available tools automatically via MCP's tool-discovery protocol.

// Claude Desktop mcp_servers config
{
"wpcodex": {
"url": "https://yoursite.com/wp-json/wpcodex/v1/mcp",
"auth": { "type": "basic", "credentials": "user:app-pw" }
}
}
5

Agent runs the session start protocol

On every session, the agent calls wpcodex/site-info for a live snapshot, then wpcodex/skill-list to discover stored playbooks. Agentic skills are loaded immediately — your agent already knows your site's patterns before you type the first prompt.

Frequently asked questions

Does WPCodex work on shared hosting?
Yes, as long as your host supports PHP 8.0+ and WordPress 6.9+. WP-CLI commands require CLI access, which some shared hosts restrict — but all other abilities work over HTTP.
Is it safe to expose my WordPress site to an AI agent?
The MCP server only activates when you explicitly add the constant to wp-config.php. Authentication uses WordPress Application Passwords, and all requests require manage_options capability. Use a dedicated admin account for agent access and review the Security page for best practices.
Which AI agents are compatible?
Any MCP-compatible client works: Claude Desktop, Cursor, Windsurf, Cline, Continue, and more. The protocol is an open standard, so compatibility grows as the ecosystem does.
Can I add my own custom abilities?
Yes. Register custom abilities with wp_register_ability() on the wp_abilities_api_init hook, or filter the abilities list via wp_codex_abilities. Full documentation is in the Docs.
Does it work on multisite?
WPCodex can be network-activated on multisite. Each subsite exposes its own MCP endpoint. Skills are stored per-site by default.
