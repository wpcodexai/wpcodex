Getting Started

Introduction

WPCodex gives AI agents unrestricted control over a WordPress installation via an MCP server plugin. This guide walks you through what it does, how to set it up, and how to connect your first agent.

1

What is WPCodex?

WPCodex turns your WordPress site into a fully programmable environment for AI agents. With arbitrary PHP execution, WP-CLI access, full filesystem access, and direct database query support, an agent can do anything WordPress can do.

The core idea: rather than building narrow, purpose-specific API endpoints, WPCodex exposes the full surface of WordPress as a set of MCP tools — and lets you add your own via a simple filter hook.

💡

New here? The fastest path is the Quick Start guide — it gets you connected in under 5 minutes.

2

Installation

WPCodex requires WordPress 6.9+ and PHP 8.0+. Install via the WordPress admin or WP-CLI:

# WP-CLI (recommended)
wp plugin install wpcodex --activate

# Or upload the ZIP via Admin → Plugins → Add New → Upload
⚠️

After activating, the MCP server is still off. You must add the constant below to wp-config.php before any MCP connections will work.

3

Enabling MCP

Add this constant to wp-config.php before the "That's all, stop editing!" line:

define( 'WPCODEX_ENABLE_MCP', true );

When absent or false, the plugin boots no MCP transport, registers no REST endpoints, and exposes no abilities. This is intentional — the plugin is dormant by default so activating it on a site doesn't immediately expose anything.

4

Authentication

WPCodex uses WordPress Application Passwords (built into WP 5.6+). Go to Users → Your Profile → Application Passwords, create a new one, and copy it — it's shown only once.

Pass it as HTTP Basic auth on every request:

Authorization: Basic <base64(username:application-password)>

The authenticating user must have the manage_options capability. We recommend creating a dedicated admin account specifically for your agent.

5

Connecting Agents

Your MCP endpoint is:

https://yoursite.com/wp-json/wpcodex/v1/mcp

For Claude Desktop, edit claude_desktop_config.json:

{
  "mcpServers": {
    "wpcodex": {
      "url": "https://yoursite.com/wp-json/wpcodex/v1/mcp",
      "headers": { "Authorization": "Basic <base64-credentials>" }
    }
  }
}
Core Abilities

Abilities Reference

All 10 core abilities are registered automatically when MCP is enabled. No configuration required.

php-execute

Execute arbitrary PHP inside the live WordPress process. Pass code (no opening tag). Use return to send data back.

wpcli-run

Run a WP-CLI command. Pass command without the leading wp. Optional timeout in seconds.

db-query

Run SQL via $wpdb. Pass sql and optional args for prepared placeholders. SELECT returns rows; mutations return affected count.

file-read / file-write

file-read returns file contents by absolute path. file-write atomically writes content to path, creating a .bak backup first.

site-info

Full install snapshot. No arguments. Returns WP version, PHP version, active plugins, active theme, and site URL. Call this at session start.

option-get / option-set

Read and write WordPress options. Complex values are auto-serialized. option-set accepts optional autoload boolean.

post-query

Run a WP_Query. Pass any query_args object. Returns found_posts count and a posts array.

Skills System

Skills

Skills are Markdown playbooks stored in the WordPress database. They give your agent persistent, site-specific knowledge that survives context resets.

1

Creating a Skill

Call wpcodex/skill-create with a name, description, and Markdown body:

// wpcodex/skill-create
{
  "name": "site-structure",
  "description": "Custom post types, taxonomies, and theme on this site.",
  "body": "---\nenable_agentic: true\n---\n\n# Site Structure\n\nTheme: Astra Pro\n...",
  "enable_agentic": true
}
2

Agentic Skills

Skills with enable_agentic: true are your agent's long-term memory. At the start of every session your agent should:

  • Call wpcodex/skill-list to retrieve all skill names and descriptions
  • For every skill where enable_agentic is true, call wpcodex/skill-read to load the full body
✅

The CLAUDE.md in the WPCodex repo contains a Session Start Protocol that implements this pattern. Copy it into your agent's system prompt.

Extension

Custom Abilities

Register your own MCP tools using the WordPress Abilities API.

1

Registering an ability

add_action( 'wp_abilities_api_init', function(): void {
  wp_register_ability( 'myplugin/my-ability', [
    'label' => __( 'My Ability', 'myplugin' ),
    'description' => __( 'What it does.', 'myplugin' ),
    'input_schema' => [ 'type' => 'object', 'properties' => [] ],
    'execute_callback' => static fn( array $args ): string => 'result',
    'permission_callback' => static fn(): bool => current_user_can( 'manage_options' ),
    'meta' => [ 'mcp' => [ 'public' => true, 'type' => 'tool' ] ],
  ] );
} );
2

Abilities Filter

Merge additional abilities using the wp_codex_abilities filter:

add_filter( 'wp_codex_abilities', function( array $abilities ): array {
  return array_merge( $abilities, [ new MyAbility() ] );
} );
3

Hooks Reference

  • wpcodex_after_activate — fires after plugin activation
  • wpcodex_before_execute — fires before any ability executes
  • wpcodex_after_execute — fires after execution completes
  • wp_codex_abilities — filter to add or remove registered abilities
Reference

Requirements

  • WordPress 6.9+
  • PHP 8.0+ (8.2+ recommended)
  • WordPress Application Passwords (default in WP 5.6+)
  • REST API accessible (not blocked by a WAF or security plugin)
  • WP-CLI installed on server (optional — only needed for wpcli-run)
