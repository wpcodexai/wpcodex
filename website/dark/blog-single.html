Building your first custom WPCodex ability

WPCodex ships with 10 core abilities out of the box — but the real power comes from extending it. The wp_codex_abilities filter lets any plugin or theme register its own abilities, exposing entirely new tools to your AI agent without touching the WPCodex core.

In this tutorial we'll build a myplugin/send-slack ability from scratch. By the end your agent will be able to call it directly, passing a channel and message string, and WPCodex will deliver it to Slack via the API.

Abilities are just PHP callables registered with the WordPress Abilities API. They live in your plugin, they have full access to WordPress, and they're automatically exposed over MCP the moment they're registered.

Prerequisites

Before you start, make sure you have:

  • WPCodex installed and MCP enabled via WPCODEX_ENABLE_MCP
  • A Slack incoming webhook URL (create one in your Slack app settings)
  • PHP 8.0+ and WordPress 6.9+
  • A plugin or functions.php to register your ability in

Anatomy of an ability

Every ability is registered with wp_register_ability() and takes a name (in namespace/kebab-case form) plus a config array. The key fields are:

  • label / description — shown in the MCP tool list and admin UI
  • input_schema — a JSON Schema object describing what parameters the ability accepts
  • execute_callback — the PHP callable that runs when the agent calls the ability
  • permission_callback — a boolean check run before execution (always require manage_options at minimum)
  • meta.mcp — tells the MCP adapter to expose this ability as a public tool

Registering the ability

Create a new file in your plugin — say includes/Abilities/SendSlack.php — and add the following:

add_action( 'wp_abilities_api_init', function(): void {
  wp_register_ability( 'myplugin/send-slack', [
    'label' => __( 'Send Slack Message', 'myplugin' ),
    'description' => __( 'Post a message to a Slack channel.', 'myplugin' ),
    'category' => 'myplugin',
    'input_schema' => [
      'type' => 'object',
      'properties' => [
        'channel' => [ 'type' => 'string', 'description' => 'Channel name, e.g. #general' ],
        'message' => [ 'type' => 'string', 'description' => 'The message text to post.' ],
      ],
      'required' => [ 'channel', 'message' ],
    ],
    'execute_callback' => 'myplugin_send_slack',
    'permission_callback' => static fn(): bool => current_user_can( 'manage_options' ),
    'meta' => [ 'mcp' => [ 'public' => true, 'type' => 'tool' ] ],
  ] );
} );

Writing the execute callback

The callback receives the validated $args array and should return a string result. Keep the function focused — validation has already been handled by the input schema, so you can trust the types:

function myplugin_send_slack( array $args ): string {
  $webhook = get_option( 'myplugin_slack_webhook' );
  if ( ! $webhook ) {
    return 'Error: Slack webhook URL not configured.';
  }

  $response = wp_remote_post( $webhook, [
    'headers' => [ 'Content-Type' => 'application/json' ],
    'body' => wp_json_encode( [
      'channel' => $args['channel'],
      'text' => $args['message'],
    ] ),
  ] );

  if ( is_wp_error( $response ) ) {
    return 'Error: ' . $response->get_error_message();
  }

  return 'Message sent to ' . $args['channel'];
}

Error handling best practices

Always return a descriptive string, even on failure — your agent reads the return value to decide what happened. Never throw exceptions from an execute callback; catch them and return the error message instead.

Testing with Claude

Restart Claude Desktop after registering the ability. In a new conversation, ask:

"Call myplugin/send-slack with channel #general and message 'Hello from WPCodex!'"

If everything is wired up correctly, Claude will call the ability and report back the result string. Check your Slack channel to confirm the message arrived.

Documenting your ability with a Skill

Once the ability is working, document it with a WPCodex Skill so your agent always knows the webhook is configured and what the ability is for:

// wpcodex/skill-create
{
  "name": "slack-ability",
  "description": "Slack webhook is configured. Use myplugin/send-slack to post messages.",
  "enable_agentic": true,
  "body": "# Slack ability\n\nWebhook is live. Use channel names like #general, #dev."
}
